# MLOps And Deployment Module Map

This map explains the new numbered path inside `14-mlops_and_deployment`.

## Main Learning Path

- `01-SQL-and-Analytics-for-AI-ML`  
  SQL patterns, warehousing intuition, analytics habits, and data checks that support ML systems.

- `02-Data-Engineering-for-ML`  
  Batch and streaming pipelines, feature tables, data contracts, and ingestion reliability.

- `03-Notebook-to-Package-Testing-and-CI`  
  Refactoring notebooks into importable code, adding tests, and wiring CI for confidence.

- `04-Model-Registry-and-Experiment-Tracking`  
  Tracking runs, comparing artifacts, promoting models, and organizing model lifecycle decisions.

- `05-Model-Serving-and-Deployment`  
  Batch inference, online inference, APIs, containers, rollout strategy, and serving tradeoffs.

- `06-Monitoring-and-Observability`  
  Logging, metrics, traces, drift checks, alerting, and runtime diagnostics.

- `07-Reproducibility-and-Pipeline-Reliability`  
  Versioning, deterministic runs, config discipline, pipeline recovery, and rollback thinking.

## Legacy And Imported Example Areas

- Existing unnumbered folders remain as example repos, experiments, or imported project snapshots.
- Use them when you want runnable reference implementations after the numbered topic overview makes sense.
