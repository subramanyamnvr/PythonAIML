# Projects Portfolio Order

This folder is now arranged by portfolio weightage, not by topic, source, or upload order.

Lower number = higher profile value.

## Ranking Logic

- end-to-end execution and deployability
- current relevance for AI, ML, GenAI, CV, and MLOps roles
- strength of the business or domain story
- interview value: architecture, tradeoffs, metrics, and demo potential
- uniqueness over very common starter projects

## How To Use This Folder

1. Start from `01` and work downward.
2. For each project, add your own explanation for the problem, data, pipeline, model choice, metrics, deployment path, and demo flow.
3. Prioritize the first `10` to `15` folders for resume bullets, LinkedIn posts, GitHub pinning, and interview walkthroughs.
4. Treat the last few folders as demos, templates, placeholders, or support assets unless you plan to expand them.

## Priority Bands

- `01` to `10`: flagship projects
- `11` to `25`: strong supporting portfolio projects
- `26` to `40`: useful demo and specialization projects
- `41` to `51`: starter, scraper, template, placeholder, or asset folders

See [project_priority.md](project_priority.md) for the full ranking and rename map.

`MLProjects.md` is kept as a legacy external resource list. The numbered folders are now the actual local run order.
