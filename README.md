# Python + AI/ML/Deep Learning/GenAI Repo

This repository is a single study and revision workspace for Python, math, data libraries, NLP, machine learning, deep learning, typed Python and Pydantic, computer vision, GenAI, MLOps, projects, and interview prep.

- numbered notebooks are the main entry point where a flat layer exists
- original support folders stay in place when notebooks depend on local data, helpers, or models
- newer topic additions use clear numbered folders or section hubs so the repo can keep growing without losing navigability

## Main Folders

- `00_repo_guide`: repo navigation docs and study planning
- `01-python-deepdive-main`: Python deep-dive notebooks with flat numbered entry notebooks
- `02-Numpy`: NumPy notebooks
- `03-Pandas`: Pandas notebooks
- `03_math_for_ai_ml`: math notes and structure for AI/ML foundations
- `04-Matplotlib`: plotting notebooks
- `05-Seaborn`: seaborn notebooks
- `06-NLPComplete`: flat NLP notebook path with datasets and README
- `07-LibrariesForDeepLearning`: flat notebook layer for PyTorch and TensorFlow courses, with source repos retained underneath
- `08_Machine_learning`: sorted classical ML topic folders, now expanded with time series, recommender systems, and causal inference or A/B testing starter folders
- `09-ComputerVision`: flat notebook layer for computer vision, plus retained `DATA`, YOLO support files, and new expansion folders for tracking, segmentation, OCR, and a capstone
- `10-deep_learning`: flat numbered deep learning topic folders, with datasets kept beside the notebooks that use them, now extended through reinforcement learning basics
- `11_typed_python_and_pydantic`: numbered space for typing, dataclasses, attrs, msgspec, and Pydantic validation or schema-focused work
- `12_genai`: flat numbered GenAI topic folders, now extended with PEFT, evals, guardrails, multimodal work, and agent memory or state
- `13_Agentic`: flat numbered week folders for agentic AI, with guides and setup folders kept intact for compatibility
- `14-mlops_and_deployment`: structured production path for SQL or analytics, data engineering, CI, model registry, serving, monitoring, and reproducibility, plus imported example repos
- `15_interview_prep`: structured interview hub with topic-wise folders for Python, SQL and stats, ML, DL, GenAI, system design, case studies, and revision sheets
- `16_projects`: curated numbered project showcase, ordered by portfolio value
- `Misc_resources`: reference area for books, papers, templates, and supporting material

## Notes

- In flattened folders, start with the numbered top-level notebooks rather than digging into the legacy source folders first.
- In `09-ComputerVision`, use the top-level numbered notebooks first, then use the new expansion folders for object tracking, segmentation, OCR, and capstone work.
- In `10-deep_learning`, start with the numbered topic folders, then open the notebooks inside each topic in project order.
- In `11_typed_python_and_pydantic`, use `03` through `06` for companion code samples, then use `02` and `07` through `16` as the main Pydantic path.
- In `12_genai`, start with `01` through `14`, use `15` through `19` for newer production-focused topics, and treat `20-Legacy-Neural-Network-Background` as optional background material.
- In `13_Agentic`, follow the numbered week folders in order and use `guides` and `setup` only when you need supporting context.
- In `14-mlops_and_deployment`, use the numbered folders as the main path and treat the imported project folders as runnable examples or reference implementations.
- In `15_interview_prep`, follow the numbered section folders and keep raw notes inside the topic they support.
- In `16_projects`, follow the numbered folders from lowest to highest number; lower number means higher resume and interview priority.
- Imported course content has been kept runnable by preserving local data paths and helper modules where needed.
