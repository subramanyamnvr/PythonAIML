# Repo Guide

Use this folder to understand the structure of the repository before diving into the course material.

## Files

- [repo_map.md](repo_map.md): what each top-level folder is for
- [study_plan.md](study_plan.md): a practical order for working through the repo

## Rules of Thumb

- In folders with a flat notebook layer, start from the numbered notebooks at the folder root.
- Keep imported course folders intact when they provide local datasets, helper scripts, or models.
- In `09-ComputerVision`, use the numbered notebooks first and treat the new tracking, segmentation, OCR, and capstone folders as expansion space.
- In `10-deep_learning`, use the numbered topic folders as the main path and keep the nested dataset folders only where the notebooks depend on them.
- In `11_typed_python_and_pydantic`, use the early companion folders for typing and related-library samples, then continue into the main Pydantic modules.
- In `12_genai`, use the numbered topic folders as the primary path, use `15` through `19` for newer production-focused topics, and treat the legacy neural-network folder as optional reference material.
- In `13_Agentic`, follow the numbered week folders first, then use `guides` and `setup` as support material.
- In `14-mlops_and_deployment`, use the numbered folders as the main learning path and the imported project folders as examples.
- Keep interview notes and revision material inside the numbered topic folders in `15_interview_prep`.
- Use `16_projects` as the numbered project run queue and showcase area.
